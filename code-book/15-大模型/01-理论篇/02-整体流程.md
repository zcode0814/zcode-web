## Function Calling

解决什么问题？

- 无法感知实时数据
- 无法获取私人数据

是谁调用了外部 API？

![](https://yitiaoit.oss-cn-beijing.aliyuncs.com/img/image-20260315163658375.png)

并不是大模型直接调用，而是 function calling 模块接收参数并选择合适的外部工具函数。

- 决策层：大模型负责理解用户语义和需求判断（如识别需要天气数据）

- 协调层：function calling模块接收参数并选择合适的外部工具函数
- 执行层：开发人员编写的工具函数（如墨迹天气API工具函数）实际调用第三方API

## MCP

通过上下文协议连接大模型以外的资源，包括数据库、浏览器和API调用。

![](https://yitiaoit.oss-cn-beijing.aliyuncs.com/img/image-20260315165856355.png)

解决什么问题？

- 不同厂商（如OpenAI/阿里千问）实现互不兼容的函数调用方案
- 单个Function Calling模块难以应对海量并发需求

工作流程？

- 客户端初始化并获取服务端工具列表
- 模型通过客户端查询功能可用性
- 客户端代理执行具体功能调用
- 结果经客户端返回给模型
