## 线上 OOM 复盘

### 快速处理

- 拆除一台服务器保留现场，其他服务重启保证可用
- dump GC日志分析

### 现象分析

1. 观察日志可知系统每隔 40S 发生一次 Full GC，耗时 200 毫秒，回收以后系统老年代占用也不大，才 15M，但是新生代回收完还有 2 个 G。 初步推测是新生代数据要晋升到老年代，结果放不进去而引起的 Full GC。
2. 分析 dump 文件，找出大对象，定位代码
3. 问题定位，将文件先读入内存再下载，用户下载大文件导致内存不足，老年代也放不进去，只能 FullGC 。

> 什么时候会发生 FullGC ？
>
> - 老年代空间不足是最主要的原因
> - 显式调用 `System.GC()`
> - G1 GC的Mixed GC阶段

4. 解决方案

- 使用对象存储
- 流失读取

```Java


public class DownloadLargeFile {
    public static void main(String[] args) {
        String fileUrl = "http://example.com/largefile.zip";
        String savePath = "path/to/save/file.zip";
  
        try {
            URL url = new URL(fileUrl);
            URLConnection connection = url.openConnection();
            try (BufferedInputStream in = new BufferedInputStream(connection.getInputStream());
                 FileOutputStream out = new FileOutputStream(savePath)) {
                byte[] buffer = new byte[4096];
                int bytesRead;
                while ((bytesRead = in.read(buffer)) != -1) {
                    out.write(buffer, 0, bytesRead);
                }
            }
            System.out.println("File downloaded successfully.");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
} 
```
