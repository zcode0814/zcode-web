## 懒汉式（双端加锁）

```java
public class Singleton_4 {
    //volatile修饰，防止指令重排
    private static volatile Singleton_4 instance;

    private Singleton_4() {
    }

    public static Singleton_4 getInstance() {
        //第一重校验，检查实例是否存在
        if (instance == null) {
            //同步块
            synchronized (Singleton_4.class) {
                //第二重校验，检查实例是否存在，如果不存在才真正创建实例
                if (instance == null) {
                    instance = new Singleton_4();
                }
            }
        }
        return instance;
    }

}
```

**为什么要用 volatile 修饰 instance？**

- 防止指令重排
- 保证可见性

> volatile 和 Atomic 类的区别：
>
> - Atomic 类是 Java 并发包中提供的一组**原子性操作类**，提供了一些原子性的操作方法，如 compareAndSet、getAndIncrement 等。
> - volatile 适用于标记变量，保证了变量的可见性和禁止指令重排，但不能保证复合操作的原子性。

## 懒汉式（CAS）

一种轻量级的线程安全的加锁方式，但是空转等待是会导致 cpu 占用过高。

```java
public class Singleton {
    private static final AtomicReference<Singleton> INSTANCE = new AtomicReference<>();

    private Singleton() {
    }

    public static Singleton getInstance() {
        Singleton currentInstance = INSTANCE.get();
        if (currentInstance == null) {
            Singleton newSingleton = new Singleton();
            if (INSTANCE.compareAndSet(null, newSingleton)) {
                return newSingleton;
            } else {
                return INSTANCE.get();
            }
        }
        return currentInstance;
    }
}

    
	
```

## 枚举

```java
public enum Singleton_7 {

    //定义一个枚举，代表了Singleton的一个实例
    INSTANCE;
    public void anyMethod(){
        System.out.println("do any thing");
    }
}

// 调用
@Test
void anyMethod() {
    Singleton_7.INSTANCE.anyMethod();
}
```
